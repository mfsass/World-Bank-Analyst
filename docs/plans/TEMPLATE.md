# Plan Template

## Goal

One sentence describing the outcome.

## Context

- Why this work exists.
- Constraints from the brief, architecture, or design system.

## Affected Areas

- Files or modules expected to change.
- Skills or docs that must be consulted.

## Implementation Steps

1. First concrete step.
2. Second concrete step.
3. Validation and cleanup.

## Validation

- Tests to run.
- Manual checks to perform.
- Risks to watch during review.

## ADR Check

- Does this change choose between viable alternatives?
- If yes, add or update an entry in [docs/DECISIONS.md](../DECISIONS.md).

## Open Questions

- Unknowns that must be resolved before or during implementation.
